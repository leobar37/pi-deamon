# T-001: Backend — Endpoints de modelos

## Objetivo
Agregar al backend del dashboard la capacidad de listar modelos disponibles y cambiar el modelo de una sesión.

## Archivos a modificar

### 1. `packages/dashboard/src/session/live-session.ts`
- Agregar métodos públicos:
  - `getModel(): { provider: string; id: string; name: string } | undefined` — retorna info del modelo actual del `_agentSession`
  - `setModel(provider: string, modelId: string): Promise<void>` — resuelve modelo vía `ModelRegistry`, valida auth, llama `_agentSession.setModel(model)`
- `getAvailableModels()` puede no ser necesario si `SessionHost` tiene acceso al `ModelRegistry`

### 2. `packages/dashboard/src/session/host.ts`
- Agregar `getModelRegistry()` o similar, o métodos proxy:
  - `getSessionModel(sessionId): ModelInfo | undefined`
  - `setSessionModel(sessionId, provider, modelId): Promise<void>`
  - `getAvailableModels(): ModelInfo[]`
- El `SessionHost` ya crea/inyecta el `AgentSession`, por lo que debería tener acceso al `ModelRegistry` usado en `createAgentSession()`

### 3. `packages/dashboard/src/procedures/schemas.ts`
- Agregar schemas Zod:
  ```ts
  export const ModelInfoSchema = z.object({
    provider: z.string(),
    id: z.string(),
    name: z.string(),
    api: z.string(),
    reasoning: z.boolean(),
  });
  ```

### 4. `packages/dashboard/src/procedures/session.ts`
- Agregar sub-router `models` dentro del objeto retornado:
  - `models.list`: input opcional `{ sessionId?: string }`, output `{ models: ModelInfoSchema[], current?: ModelInfoSchema }`
  - `models.set`: input `{ sessionId: string, provider: string, modelId: string }`, output `{ success: boolean }`

### 5. `packages/dashboard/src/procedures/index.ts`
- Verificar que el router incluye los nuevos procedimientos (debería ser automático si se agregan al objeto retornado)

## Notas de implementación
- `ModelRegistry` viene de `@earendil-works/pi-coding-agent` — verificar cómo se importa actualmente en el backend
- Los modelos a listar deben ser los `getAvailable()` (filtrados por auth configurada)
- `AgentSession.setModel()` ya maneja la emisión del evento SSE `model_select`
- Usar el logger existente para loggear cambios de modelo
- Mantener el estilo de error handling existente (wrap en `Error('...')`)
