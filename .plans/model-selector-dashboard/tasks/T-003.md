# T-003: Frontend — UI Selector de Modelos

## Objetivo
Crear e integrar un componente selector de modelos en el `ChatHeader`.

## Archivos a modificar/crear

### 1. `packages/dashboard/frontend/src/components/ModelSelector.tsx` (nuevo)
- Dropdown/select que muestra el modelo actualmente activo
- Al hacer click/focus, carga modelos disponibles vía `orpc.sessions.models.list({ sessionId })`
- Agrupa modelos por `provider` usando `<optgroup>` o secciones custom
- Al seleccionar uno, llama `orpc.sessions.models.set({ sessionId, provider, modelId })`
- Estados de loading y error
- Estilo con Tailwind, consistente con el resto del dashboard (text-xs, bordes sutiles, dark theme)

### 2. `packages/dashboard/frontend/src/components/ChatHeader.tsx`
- Importar y renderizar `<ModelSelector sessionId={sessionId} />` en el área derecha que actualmente está vacía:
  ```tsx
  {/* Right: status-only area, no controls */}
  <div className="flex items-center gap-2 shrink-0" />
  ```
- Reemplazar ese div vacío por el selector

### 3. `packages/dashboard/frontend/src/components/ChatInput.tsx` (opcional)
- Considerar mostrar un indicador sutil del modelo activo cerca del input (fuera de scope si no se pide)

## Notas de diseño
- El selector debe ser compacto porque el header tiene altura fija (`h-14`)
- Usar un `<select>` nativo o un dropdown custom con Tailwind
- El nombre del modelo puede ser largo, usar `truncate` y `max-w-*`
- Considerar mostrar el provider como prefixo o tooltip (ej: "Anthropic / Claude Opus 4")
