# T-003: Rewrite `SubAgentEventBus`

**Phase**: subagents  
**Dependencies**: T-002  
**Requirements**: FR-003, NFR-005

## Objective

Rewrite `packages/subagents/src/event-bus.ts` so `SubAgentEventBus` uses `TypedEventBus<typeof SubAgentEvents>` internally.

## API Changes

**New primary API**:
- `publish(creator, payload)` — type-safe publish
- `subscribe(creator, handler)` — typed subscription

**Legacy API (kept for migration)**:
- `on(type, handler)` → delegates to `inner.on(...)`  with string matching
- `emit(event)` → delegates to `inner.publish()` via a reverse-lookup from type string to creator
- `off(type, handler)` → delegates to `inner.off()`
- `clear()` → delegates to `inner.clear()`

**Internal changes**:
- The bus now emits events with `timestamp` and `id` (auto-generated by `TypedEventBus`)

## Resulting Event Object

The new `TypedEvent` adds `id` and wraps payload. Backward-compatible `emit()` strips `id` and flattens payload back into the top-level object, so legacy listeners see the same shape.

## Files to Edit

- `packages/subagents/src/event-bus.ts` (rewrite)

## Verification

- Legacy `on("task.start", handler)` + `emit({ type: "task.start", ... })` still works
- New `subscribe(SubAgentEvents.taskStart, handler)` + `publish(SubAgentEvents.taskStart, payload)` works
- Wildcard `.on("*", handler)` still receives all events
- `clear()` removes all listeners
