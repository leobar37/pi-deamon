# T-003: Unify Execution Systems

## Objective
Remove the dual execution system (System A: `src/execution/` vs System B: `TaskExecutor`). Keep only `TaskExecutor` as the single execution path.

## Problem
- System A (`src/execution/chain.ts`, sequential.ts, parallel.ts, execute.ts): 5 files, duplicate strategy logic
- System B (`src/task-executor.ts`): used by Lion runtime, supports abort, event forwarding
- `controller.executePlan()` calls System A (src/controller.ts:144-152)
- Chain.ts line 30 has type-unsafe `(instance as unknown as { task: DelegationTask }).task = task` mutation
- Code paths diverge: Lion tools use TaskExecutor, extension tools use controller + System A

## Changes

### Modify `src/controller.ts`
- Replace `executePlan()` implementation: use `TaskExecutor` internally instead of System A functions
- Import `TaskExecutor` from `src/task-executor.ts`
- Create a `TaskExecutor` with `{ controller: this, onEvent: emit }` and call `executor.execute(plan)`
- Keep the same return type `Promise<DelegationResult[]>`

### Update imports
- Remove imports from `src/execution/index.ts` (already deleted in T-001)
- Add import for `TaskExecutor`

### Update `src/task-executor.ts` (if needed)
- Ensure `TaskExecutor` supports all execution plan shapes that `executePlan()` previously supported
- Verify abort path works correctly

### Update tests
- Move/update tests from `test/execution/chain.test.ts`, `test/execution/sequential.test.ts`, `test/execution/parallel.test.ts` into `test/task-executor.test.ts`
- The execution tests are unit tests - they test strategy logic with fake instances. Adapt them to TaskExecutor's interface.

## Files Affected
- Modify: `src/controller.ts` (use TaskExecutor)
- Modify: `src/task-executor.ts` (ensure full strategy support)
- Modify: `test/task-executor.test.ts` (incorporate strategy tests)
- Delete: `test/execution/` directory (redundant tests after migration)

## Acceptance Criteria
- `controller.executePlan()` uses `TaskExecutor` internally
- All three strategies (sequential, parallel, chain) work through TaskExecutor
- Type-unsafe mutation at chain.ts:30 is gone (chain.ts is deleted)
- `controller.ts` no longer imports from `src/execution/`
- All tests pass

## Validation
1. Run controller.test.ts (uses executePlan with mocks)
2. Run task-executor.test.ts (updated with strategy tests)
3. Run lion workflow test (uses TaskExecutor)
4. Verify chain strategy still passes task context between steps
