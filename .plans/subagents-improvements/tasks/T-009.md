# T-009: Dashboard Reconstruction

## Objective
Fix the dashboard so it properly reconstructs state from persisted events on start, and handles `session_tree` branch switches.

## Problems
1. `LionDashboardServer` is created fresh on each `start()` - no event replay (dashboard.ts:20-48)
2. Dashboard NOT restarted on `session_tree` (index.ts:43)
3. `restoreLoneCore()` has no chronological ordering (core.ts:143-152)
4. Dashboard cannot start without active controller (dashboard.ts:21-23) - ungraceful error

## Changes

### Event replay on dashboard start
- When `LionDashboardServer.start()` is called, load all persisted events from `SubAgentEventStore`
- Replay them through the state manager to rebuild virtual instances
- Emit them to connected SSE clients
- Then subscribe to live events as before

### session_tree handling
- In `src/lion/index.ts` (or wherever `session_tree` handler is), call `runtime.restore(ctx)` AND `runtime.ensureDashboard(ctx)` 
- Dashboard server checks if it's already running before creating a new one

### Chronological ordering in restoreLoneCore
- Sort entries by timestamp when restoring instead of relying on iteration order
- Add `timestamp` to the persisted entry schema if missing

### Better dashboard start error
- If no active controller, show meaningful message instead of throwing
- Or: start dashboard anyway with only virtual instances (reconstructed from persisted events)

## Files Affected
- Modify: `src/lion/dashboard.ts` (event replay on start)
- Modify: `src/lion/index.ts` (session_tree handler, ensureDashboard call)
- Modify: `src/lion/core.ts` (chronological restore ordering)
- Modify: `src/transport/state-manager.ts` (expose replay method if needed)
- Modify: `src/transport/event-store.ts` (expose readAllEvents method if needed)

## Acceptance Criteria
- Dashboard restarted on `session_tree` after branch switch
- Dashboard start replays past events to SSE clients
- Dashboard start does not require active controller (shows historical data)
- `restoreLoneCore()` sorts by timestamp

## Validation
1. Start session, create subagent activity, switch session tree, verify dashboard shows events
2. Kill and restart dashboard, verify historical events appear
3. Start dashboard without active Lion controller, verify it shows empty state (not crash)
