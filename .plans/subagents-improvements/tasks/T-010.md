# T-010: Fix Execution and State Machine Bugs

## Objective
Fix two concrete bugs found during analysis: the type-unsafe instance mutation in chain execution, and the missing handshake between parallel state machines.

## Bug 1: Type-Unsafe Instance Mutation (Chain Execution)

### Location
- Previously in `src/execution/chain.ts:30` (already deleted in T-001/T-003 but pattern may exist in `task-executor.ts`)
- Pattern: `(instance as unknown as { task: DelegationTask }).task = task`
- This bypasses type checking to override a private field

### Fix
- In `TaskExecutor.chain()` (src/task-executor.ts), verify there is no equivalent type-unsafe mutation
- If the pattern exists, add a proper `setTask()` method on `SubAgentInstance` instead of the cast
- Or pass `task` as a parameter to `instance.start()` instead of mutating after creation

## Bug 2: Parallel State Machine Handshake

### Location
- `LionCore` run states (src/lion/core.ts:19-27): `"executing"`, `"awaiting_orchestrator"`, `"reviewing"`, `"correcting"`, `"approved"`, `"rejected"`, `"failed"`
- `MainSessionBridge` thread states (src/lion/main-session.ts:18-24): `"running"`, `"completed"`, `"paused"`
- No handshake between them: `finishRun()` doesn't tell MainSessionBridge, bridge doesn't know about review states

### Fix
- Add a `notifyRunComplete(result: LionBuildResult)` method to `MainSessionBridge`
- Call it from `LionCore.finishRun()` (or from the caller of finishRun in runtime.ts)
- In bridge, update state to `"completed"` when notified
- Add a `getDetailedState()` method on bridge that returns the full LionCore state when bridge is connected to a runtime

### Files Affected
- Modify: `src/lion/core.ts` (add notification call)
- Modify: `src/lion/main-session.ts` (add notifyRunComplete)
- Modify: `src/lion/runtime.ts` (bridge notification wiring)
- Verify: `src/task-executor.ts` (check for type-unsafe mutation)

## Acceptance Criteria
- No type-unsafe `as unknown as` pattern for task mutation exists
- `MainSessionBridge` knows when a run completes (not just "thread ended")
- Dashboard receives run-complete event with build result
