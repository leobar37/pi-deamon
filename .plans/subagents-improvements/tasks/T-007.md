# T-007: Test Transport Layer

## Objective
Add test coverage for the transport layer: `HttpServerTransport`, `DashboardStateManager`, and `SubAgentEventStore`. These 6 files (~800 lines) currently have zero tests.

## Priority

### 1. `SubAgentEventStore` (src/transport/event-store.ts)
**Unit test**: Test append/write to JSONL files, test rotation/pruning (if added), test recovery/read.
**Why first**: Fundamental building block, no HTTP dependency.

### 2. `DashboardStateManager` (src/transport/state-manager.ts)
**Mock test**: Test live/virtual instance reconciliation, registerLiveInstance, unregisterLiveInstance, getAllInstances, reconstruction from persisted events.
**Why second**: Core state merging logic, medium complexity.

### 3. `HttpServerTransport` (src/transport/http-server.ts)
**Integration test**: Start a Bun.serve instance, connect to SSE endpoint, verify events arrive, test REST API endpoints (/api/threads, /api/threads/:id, /api/events), test static file serving.
**Why third**: Requires HTTP server setup, most complex.

## Test Approach
- Use Vitest with real Bun.serve for HTTP tests (no external process needed)
- Use mock SubAgentController for transport tests
- Use tmp directory for event store file I/O
- Test with controlled event sequences (not real subagent runs)

## Files to Create
- `test/transport/event-store.test.ts`
- `test/transport/state-manager.test.ts`
- `test/transport/http-server.test.ts`

## Acceptance Criteria
- Event store: write events, read back, handle file rotation, edge cases (empty, concurrent writes)
- State manager: live instance registration/deregistration, virtual instance reconstruction, getAllInstances merge order
- HTTP server: SSE event delivery, REST API returns correct data, static files served, CORS headers present
- All tests pass in isolation (no external dependencies)
