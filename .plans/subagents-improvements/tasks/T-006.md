# T-006: Fix Type System Issues

## Objective
Fix type system violations: replace inline `import()` expressions, derive `EffectiveSubAgentConfig` from `SubAgentDefinition`, clean up leaky re-exports.

## Problems
1. `src/types.ts` lines 202-205, 242, 250, 263: Inline `import("./transport/types.js")` expressions (violates AGENTS.md rule)
2. `src/types.ts` lines 72-88 vs 96-112: `EffectiveSubAgentConfig` is a hand-copied duplicate of `SubAgentDefinition`
3. `src/types.ts` lines 170-174: `authStorage`, `modelRegistry`, `settingsManager` typed as optional but required at runtime
4. `src/types.ts` lines 270-275: Re-exports 3 external package types (leaky abstraction)

## Changes

### Fix inline imports (Problem 1)
Replace all `import("./path.js").Type` inline expressions with proper top-level imports:
```typescript
// Before (types.ts:202)
onEvent?: (event: import("./transport/types.js").SubAgentEvent) => void;

// After
import type { SubAgentEvent } from "./transport/types.js";
// ... then:
onEvent?: (event: SubAgentEvent) => void;
```

### Derive EffectiveSubAgentConfig (Problem 2)
Change `EffectiveSubAgentConfig` to be derived from `SubAgentDefinition`:
```typescript
// Instead of hand-copied fields:
export type EffectiveSubAgentConfig = 
  // All fields from SubAgentDefinition are inherited
  // Specific overrides typed inline
```

### Fix optional types (Problem 3)
Make `authStorage`, `modelRegistry`, `settingsManager` required in the config interface, or add proper runtime guards in `session-factory.ts` to handle missing values with clear error messages.

### Remove leaky re-exports (Problem 4)
Stop re-exporting `ExtensionAPI`, `AgentMessage`, `ImageContent` from external packages. Consumers should import from the owning packages directly.

## Files Affected
- Modify: `src/types.ts` (all 4 fixes)

- Modify: `src/session-factory.ts` (add runtime guards for optional config)
- Modify: `src/index.ts` (remove leaky re-exports)
