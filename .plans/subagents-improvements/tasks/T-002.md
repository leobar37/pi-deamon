# T-002: Unify Event Shape

## Objective
Remove the `createEvent()` factory wrapper and its `TypedEvent<type, payload>` shape. All events use a single flat shape matching `SubAgentEventMap`/`LionEventMap` directly. Eliminate the unsafe `as LionEvent` cast in `LionRuntimeEventBus.publish()`.

## Problem
- `createEvent()` (src/event-core.ts:27-42) wraps payload in `.payload` property
- Neither `SubAgentEventBus` nor `LionRuntimeEventBus` consumes the wrapper shape - both flatten at emit time
- `LionRuntimeEventBus.publish()` (src/lion/events/bus.ts:8-14) does runtime spread + unsafe `as LionEvent` cast
- Auto-generated `id` from `createEvent()` is thrown away (flat events don't have an `id` field)
- `src/event-defs.ts` maintains 9 event creators that are transpiled away (zero runtime value)
- Double-specification: payload types in both `types.ts` event maps AND `event-defs.ts` creator type params

## Changes

### Remove
- `src/event-defs.ts` (entire file - all event creators)
- `src/event-core.ts` function `createEvent()` (keep file for now if other utilities exist, or delete entire file)

### Modify
- `src/event-bus.ts` - inline `EventBusBase` into `SubAgentEventBus` since the abstraction adds no value for a single subclass
- `src/lion/events/bus.ts` - remove `publish()`, emit flat events directly via a simpler method
- `src/lion/events/defs.ts` - replace `createEvent(...)` calls with plain type-annotated factory objects or remove entirely (move type defs to types.ts)
- `src/lion/events/types.ts` - remove if it's a pure re-export
- `src/lion/events/index.ts` - update re-exports
- All emit sites in `instance.ts`, `controller.ts`, `lion/runtime.ts` - emit flat objects directly instead of using creators

### Type simplification
- `src/types.ts` - `SubAgentEventMap` becomes the single source of truth for SubAgent events
- `src/lion/types.ts` - `LionEventMap` becomes the single source of truth for Lion events  
- Remove duplicate shape spec in `src/lion/events/defs.ts`
