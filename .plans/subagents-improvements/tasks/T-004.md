# T-004: Simplify Delegation Guard

## Objective
Reduce `src/lion/delegation-guard.ts` from ~113 lines to ~15 lines by removing fragile intent detection, per-turn probe budgets, and verbose block messages.

## Problem
- Complex regex-based intent detection (`isNonTrivialRepositoryWork`, delegation-guard.ts:61-67) is fragile
- Spanish coverage is incomplete and language-locked
- Per-turn probe budget (`STRUCTURE_PROBE_BUDGET = 3`, delegation-guard.ts:50) resets every turn, allowing indefinite probing across multiple turns
- Verbose block messages (delegation-guard.ts:37-46) waste tokens
- `isPlanFileToolCall` (delegation-guard.ts:69-75) iterates all input keys O(n) per tool call
- Guard is registered unconditionally (index.ts:57), running logic even when Lion is inactive

## Changes

### Simplified guard logic
When Lion is active:
- Allow: all plan tools (`lion_list_plans`, `lion_activate_plan`, `lion_reconcile_plan`)
- Allow: tool calls that touch `.plans/` paths
- Block: `edit`, `write`, `bash`, `grep` (destructive tools) with a single short message "Use lion_tasks to delegate this work"
- Allow: everything else (reads, probes, finds) - the orchestrator prompt already tells the agent to use delegation

### Remove
- `isNonTrivialRepositoryWork()` function (whole regex logic)
- `STRUCTURE_PROBE_BUDGET` constant
- Per-turn probe counters and budget tracking
- Verbose multi-line block messages
- `isPlanFileToolCall()` with full iter (just check first arg or use a simple path check)
- `collectInputPaths()` function

### Guard registration
- Register the hook only when Lion is activated (not unconditionally at index.ts:57)
- Use a registration method on LionRuntime that hooks/unhooks on activate/deactivate

## Files Affected
- Modify: `src/lion/delegation-guard.ts` (simplify to ~15 lines)
- Modify: `src/lion/index.ts` or `src/lion/runtime.ts` (conditional registration on activate)
- Modify: `src/lion/types.ts` (remove guard-related types if any)

## Acceptance Criteria
- `delegation-guard.ts` is ~15-20 lines
- No regex intent detection
- No probe budget
- Block messages are one line
- Guard only runs when Lion is active
- All existing guard tests pass (test/lion/workflow.test.ts)

## Validation
1. Verify guard blocks `edit`/`write`/`bash`/`grep` when Lion active
2. Verify guard allows `lion_*` tools when Lion active
3. Verify guard allows `read_file` when Lion active
4. Verify guard does nothing when Lion inactive
5. Run workflow test
