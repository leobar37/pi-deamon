## Patrón de testing

Cada test sigue el patrón AAA (Arrange-Act-Assert):

```typescript
it("sends a message on enter", async () => {
  // Arrange
  const sessionId = "test-session";
  const user = userEvent.setup();
  renderWithProviders(<Composer sessionId={sessionId} />);

  // Act
  const textarea = screen.getByPlaceholderText("Type a message...");
  await user.type(textarea, "Hello, world!{Enter}");

  // Assert
  // Verify message was sent (check mock was called, or check optimistic message appeared)
});
```

Para tests que necesitan un runtime con datos pre-poblados:

```typescript
it("renders existing messages", () => {
  // Arrange: create runtime, add messages
  const runtime = createSessionRuntime();
  const sessionId = "test-session";

  // Add session to runtime
  runtime.store.set(runtime.maps.sessions.mapAtom, {
    type: "set",
    key: sessionId,
    value: mockSessionEntry({ info: { ...mockSessionInfo(), id: sessionId } }),
  });

  // Add messages
  const userMsg = mockUserMessage({ sessionId, blocks: [{ type: "text", text: "Hello" }] });
  const asstMsg = mockAssistantMessage({ sessionId, blocks: [{ type: "text", text: "Hi there!" }] });
  runtime.store.set(runtime.maps.messages.mapAtom, { type: "set", key: userMsg.id, value: userMsg });
  runtime.store.set(runtime.maps.messages.mapAtom, { type: "set", key: asstMsg.id, value: asstMsg });

  // Act
  renderWithProviders(<ChatView sessionId={sessionId} />);

  // Assert
  expect(screen.getByText("Hello")).toBeTruthy();
  expect(screen.getByText("Hi there!")).toBeTruthy();
});
```

## Notas
- Usar `screen.getByText`, `screen.getByPlaceholderText`, `screen.getByRole` para queries
- Usar `userEvent` en lugar de `fireEvent` para simular interacciones realistas
- Para verificar llamadas a API mock, usar spies en el mock client
- Los tests deben ser independientes y aislados (cada test crea su propio runtime)
- No mockear el runtime — usar `createSessionRuntime()` real para tests de integración
- El runtime real permite probar la lógica de eventos SSE y optimistic UI
