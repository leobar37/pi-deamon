# T-003: Refactor ChatInput → Composer modular

## Objetivo
Dividir `ChatInput.tsx` (~180 líneas) en una estructura de módulos `composer/` con responsabilidades claras y separadas. `ChatInput` debe seguir exportándose para backward compatibility.

## Archivos a crear

### `src/components/composer/Composer.tsx`
Orquestador principal:
- Estado del texto (`text`, `setText`)
- Estado de `sending`
- Lógica de submit (`handleSubmit`) con manejo de steer/prompt
- Lógica de abort (`handleAbort`)
- Renderiza: `ComposerIndicator` → `ComposerQueueChips` → `ComposerToolbar` → (textarea + actions en fila)
- Props: `sessionId: string`

```tsx
export function Composer({ sessionId }: { sessionId: string }) {
  const [text, setText] = useState("");
  const [sending, setSending] = useState(false);
  const { isStreaming, pendingSteering, pendingFollowUp } = useSessionStreaming(sessionId);
  const runtime = useSessionRuntime();

  const actions = useMemo(() => {
    const optimistic = createOptimisticManager(runtime);
    return createActions(runtime, optimistic);
  }, [runtime]);

  const handleSubmit = useCallback(async () => { ... }, [text, sending, isStreaming, sessionId, actions]);
  const handleAbort = useCallback(async () => { ... }, [sessionId, actions]);

  return (
    <div className="bg-bg-base border-t border-border-subtle px-4 py-4">
      {isStreaming && <ComposerIndicator />}
      {(pendingSteering.length > 0 || pendingFollowUp.length > 0) && (
        <ComposerQueueChips steering={pendingSteering} followUp={pendingFollowUp} />
      )}
      <div className="relative flex items-end gap-2 bg-bg-elevated border border-border-default rounded-xl px-3 py-2 ...">
        <ComposerToolbar sessionId={sessionId} />
        <ComposerTextarea value={text} onChange={setText} onKeyDown={handleKeyDown} disabled={sending} />
        <ComposerActions
          hasText={text.trim().length > 0}
          sending={sending}
          isStreaming={isStreaming}
          onSubmit={handleSubmit}
          onAbort={handleAbort}
        />
      </div>
    </div>
  );
}
```

## Archivos a modificar

### `src/components/ChatInput.tsx`
Convertir en un wrapper que importa y re-exporta `Composer`:
```typescript
export { Composer as ChatInput } from "./composer/Composer.js";
```
O alternativamente:
```typescript
import { Composer } from "./composer/Composer.js";
export const ChatInput = Composer;
```
La idea es que `ChatView.tsx` siga funcionando sin cambios.

## Notas de diseño
- Cada sub-componente recibe solo las props que necesita (sin prop drilling)
- El textarea ref se maneja internamente en `ComposerTextarea` (puede exponerse via `forwardRef` si necesario)
- Los estilos deben ser idénticos al ChatInput actual (`bg-bg-elevated`, `rounded-xl`, focus ring, etc.)
- No agregar nuevas funcionalidades — solo refactor
- `bun run build` y `bun run test` deben pasar
